commit log 规范: <type>(<scope-subscope>): <subject>

# type
类型，必须是以下之一：

| 类型 | 使用场景 | 示例 |
|------|----------|------|
| **feat** | 新增功能或特性 | 添加用户登录功能、新增 API 接口 |
| **fix** | 修复 bug | 修复登录失败问题、修复空指针异常 |
| **perf** | 性能优化 | 优化查询速度、减少内存占用 |
| **refactor** | 代码重构（不改变功能） | 重构用户服务类、提取公共方法 |
| **style** | 代码格式调整（不影响逻辑） | 格式化代码、调整缩进、删除多余空行 |
| **test** | 测试相关 | 添加单元测试、补充测试用例 |
| **docs** | 文档更新 | 更新 README、添加 API 文档 |
| **chore** | 杂项/构建过程/辅助工具 | 更新依赖版本、修改配置文件 |
| **ci** | 持续集成相关 | 修改 GitHub Actions、调整 CI 配置 |
| **build** | 构建系统或外部依赖 | 修改 webpack 配置、更新构建脚本 |

# scope
影响范围，[可选]

# subject
使用简体中文，简短描述变更内容，不超过 50 个字符，使用动词开头号
