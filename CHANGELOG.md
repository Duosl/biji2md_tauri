# 更新日志

本文档记录当前正式版本的重要变化。1.0.0 之前的历史版本已归档到 [CHANGELOG_ARCHIVE.md](CHANGELOG_ARCHIVE.md)。

## biji2md 1.0.0 - 2026-05-15

### 正式版能力

- 提供主笔记与子笔记批量同步导出能力，可将云端笔记保存为本地 Markdown 文件。
- 支持增量同步和全量同步：日常同步优先处理新增内容，必要时可重新检查全部笔记。
- 支持按知识库、按月份、按标签和平铺四种目录结构组织导出文件。
- 支持 Obsidian Wikilink 与标准 Markdown 两种父子笔记链接格式，方便在不同 Markdown 工具中阅读。
- 支持本地缓存重导出，修改目录结构或链接格式后，无需重新请求接口即可重新生成 Markdown。
- 支持导出到新目录或在当前目录重新导出，便于预览新结构或更新已有导出目录。
- 每个导出的 Markdown 文件包含标题、笔记 ID、标签、知识库、创建时间和更新时间等元数据。
- 文件名默认基于笔记标题生成，并会自动处理非法路径字符和重复标题，避免覆盖同名文件。
- 启动时提供首次使用引导，帮助完成 Token 与导出目录配置。
- 同步页面集中展示配置状态、同步模式、处理进度、本地缓存数量、上次同步结果和失败项。
- 增量同步没有新内容时会明确提示已经是最新笔记，减少误判为同步异常的情况。
- 诊断日志支持关键节点、错误和全部日志筛选，并可从应用内打开历史日志文件。
- Token 保存为独立显式动作，导出目录等普通设置支持字段级保存，配置过程更清晰。
- 不同导出目录可独立保存目录结构和父子链接格式，切换目录后会自动读取该目录偏好。
- 应用配置、缓存、索引、历史和日志保存在应用数据目录，导出目录只保留 Markdown 文件和目录级配置。
- 界面适配不同平台标题栏和窗口布局，常用操作入口在窄屏下也保持可用。
- 支持自动检查更新、后台下载新版本，并在下载完成后通过标题栏或设置页一键重启更新。
- 支持 macOS、Windows 和 Linux 桌面端安装包。

### 注意

- Token 仅保存在本机配置中，不会上传到任何服务端，也不会写入同步日志。
- 「在当前目录重新导出」会替换当前导出目录中的 biji2md 导出内容；如果目录中混有手工文件，建议先选择「导出到新目录」确认结果。
- 导出目录中仅会保留用户可见的 Markdown 文件和 `.biji2md/config.json`；同步缓存、索引、历史和日志不会混入导出目录。
